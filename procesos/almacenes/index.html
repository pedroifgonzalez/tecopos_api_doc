    Almacenes

    Los almacenes de un negocio forman parte de las áreas que se gestionan en Tecopos.

    Similar a cómo funciona un almacén físico con productos y personas encargadas de llevar un inventario, la API de Tecopos realiza los mismos flujos:


    Nota importante:

    Para completar cada uno de estos flujos , el cliente de las API de Tecopos debe estar primeramente autenticado a través de un usuario que le permita realizar dicha operación


    Gestión de Almacenes

    Limitado a usuarios de rol Administrador


    • Creación de un almacén
    • Modificación de propiedades y usos de un almacén
    • Información sobre un almacén


    Creación de un almacén

    Un almacén no es más que la creación de un área especificando que sea de tipo almacén y proveyendo un nombre para este como identificador de cara a los usuarios.

    flowchart LR
    A[Cliente autenticado] -- POST ---> B[ /administration/area]
    B -- Éxito --> C[201]
    B -- Error --> D[Errores<sup>1</sup>]

    1: consultar los posibles errores manejados por la API



    Modificación de propiedades y usos de un almacén

    De manera general se pueden modificar los atributos que son generados por parte del usuario. Los que más interesan son el nombre, el estado, y además regular el uso de movimientos directos así como el hacer despachos a partir de órdenes de producción.

    flowchart LR
    A[Cliente autenticado] -- GET<sup>1</sup> --> B[ /administration/area?type=STOCK]
    B -- PATCH --> B1
    A -- PATCH ---> B1[ /administration/area/*idArea*]
    B1 -- Éxito --> C[201]
    B1 -- Error --> D[Errores<sup>2</sup>]

    1: en caso de que no tenga el identificador del almacén desde antelación
    2: consultar los posibles errores manejados por la API



    Información sobre un almacén

    De un almacén además de su nombre y estado se pueden obtener demás tipos de informaciones como una descripción, un código de referencia, si constituye un almacén principal, y otras características comunes que poseen las áreas.

    flowchart LR
    A[Cliente autenticado] -- GET ---> B[ /administration/area/*idArea*]
    B -- Éxito --> C[200]
    B -- Error --> D[Errores<sup>1</sup>]

    1: Consultar los posibles errores manejados por la API

    Endpoints

    Almacenes (Áreas)

    Descripción Método HTTP Dirección Entrada de datos Respuesta
    Obtención de todos los almacenes GET /administration/area?type=STOCK
    		 Filtros comunes de paginado Paginado de Objeto Area de tipo almacén
    Obtención de un almacén en específco GET /administration/area/{idArea} idArea
    Creación de un nuevo almacén POST /administration/area NewArea Objeto Area de tipo almacén
    Modificación de un almacén existente PATCH /administration/area/{idArea} idArea Objeto Area de tipo almacén
    Eliminación de un almacén DELETE /administration/area/{idArea} idArea

    Parámetros:

    • idArea: identificador único de un almacén


    Productos

    Descripción Método HTTP Dirección Entrada de datos Respuesta
    Creación de un nuevo producto POST /product Objeto NewProduct
    Obtención de productos GET /product Paginado de Objeto Product
    Obtención de productos por área GET /administration/product/area/{idArea} idArea

    Objetos:

    Objeto NewProduct
    Propiedades:

    • name [string]:
      nombre del producto
    • price [number]:
      precio del producto
    • salesCategoryId [number]:
      identicador único de la categoría de ventas a la que pertenece el producto
    • type [string]:
      tipo de producto

    Ejemplo:

    {
  "name": "Pantalón",
  "price": 350,
  "salesCategoryId": 15,
  "type": "VARIATION"
}
